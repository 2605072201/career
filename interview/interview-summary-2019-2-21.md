## 留言

### 是否建议背面试题

如果其中某些题，你没有办法较为顺畅的表达出来，那说明你可能掌握的还不够，或者理解的不太顺畅。这个时候，我不太建议你通过背答案来准备面试。

我是建议再回去总结下，屡屡思路。我觉得通常理解的较为深入的情况下，应该是可以不用背的（我没有背面试题的习惯）。

也不排除有些例外，譬如，

> html5 新增了哪些 api ?

这种看起来很记忆性的题，我也建议总结归类，而不是记一大堆 api 列表。譬如：语义化标签 (`header`/`footer`/`nav`)，多媒体 (`video`/`audio`)，图形渲染 (`canvas`)，本地存储 (`localStorage`/`sessionStorage`/`IndexDB`/`webSql`)，前后通信 (`webSockets`)，性能 (`webWorker`)，，，

### 面试中的主要瓶颈

- 性能优化 / `webpack` 不熟，离中级前端入门还有些距离
- 半年经验，因为这个或多或少卡了一点薪资，面试官也不太敢发 offer，其中有一份 offer 是因为前期的候选人没去，才轮到我。
